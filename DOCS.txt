----------------------------------------------------------
  skills/gcalcli-calendar/
----------------------------------------------------------
PARA QUÉ SIRVE:
  Permite a Claw leer el Google Calendar de Seba (eventos, agenda,
  reuniones) usando la herramienta de línea de comandos gcalcli.

CUENTAS QUE CUBRE:
  - sebastian@ilitglobal.com (calendario principal, incluye PandaGo)
  - se.urquiza@gmail.com (personal)

ARCHIVO CLAVE:
  gcalcli-wrapper.sh — script bash que invoca gcalcli con las credenciales
  correctas (client_id y client_secret se cargan como variables de entorno
  GCALCLI_CLIENT_ID y GCALCLI_CLIENT_SECRET; los tokens OAuth están en
  ~/.gcalcli/).

USO TÍPICO:
  bash skills/gcalcli-calendar/gcalcli-wrapper.sh --nocolor agenda today +7d

NOTA IMPORTANTE:
  El evento "Busy" en el calendario = sesión de entrenamiento de Seba.

========================================================
  SCRIPTS DE AUTENTICACIÓN (scripts/)
========================================================

----------------------------------------------------------
  scripts/auth_gmail_calendar.py
----------------------------------------------------------
PARA QUÉ SIRVE:
  Autentica la cuenta personal se.urquiza@gmail.com con Google OAuth
  para acceso al calendario de Gmail.

GENERA:
  ~/.gcalcli/token_gmail.json

CUÁNDO USARLO:
  Solo si el token de Gmail expira o se revoca. Abre el navegador
  para que Seba autorice manualmente.

----------------------------------------------------------
  scripts/auth_drive.py
----------------------------------------------------------
PARA QUÉ SIRVE:
  Autentica sebastian@ilitglobal.com con Google OAuth para acceso
  a Google Drive (solo lectura) y Google Calendar.

GENERA:
  ~/.gcalcli/token_drive.json

PARA QUÉ USA DRIVE:
  Buscar y leer transcripts/notas de reuniones guardadas como Google Docs
  (generadas por Gemini automáticamente tras las reuniones).

CUÁNDO USARLO:
  Solo si el token de Drive expira o se revoca.

----------------------------------------------------------
  scripts/auth_work_calendar.py
----------------------------------------------------------
PARA QUÉ SIRVE:
  Autentica la cuenta de trabajo sebastian.urquiza@pandago.eco con
  Google OAuth para acceso al calendario corporativo de PandaGo.

GENERA:
  ~/.gcalcli/token_work.json

CUÁNDO USARLO:
  Solo si el token de trabajo expira o se revoca.

----------------------------------------------------------
  scripts/auth_whoop.py
----------------------------------------------------------
PARA QUÉ SIRVE:
  Autentica con la API de Whoop via OAuth2 para leer datos de salud
  y recuperación de Seba.

GENERA:
  ~/.whoop/token.json

DATOS QUE PERMITE LEER:
  - Recuperación diaria (recovery score 0-100)
  - Ciclos de strain
  - Sueño
  - Workouts
  - Perfil y medidas corporales

CUÁNDO USARLO:
  Solo si el token de Whoop expira (tiene refresh_token, normalmente
  se renueva automáticamente).

----------------------------------------------------------
  scripts/whoop_test.py
----------------------------------------------------------
PARA QUÉ SIRVE:
  Script de prueba rápida para verificar que la API de Whoop funciona.
  Muestra el perfil, la última recuperación y los últimos 3 workouts.

CUÁNDO USARLO:
  Para diagnosticar si hay problemas con el token o la API de Whoop.

NOTA:
  El token se refresca automáticamente antes de cada llamada real.
  El User ID de Seba en Whoop es: 30856155

========================================================
  NOTAS DE SEGURIDAD
========================================================

- Los tokens OAuth (Google, Whoop, Slack) están SOLO en local (~/)
  y están excluidos del repo via .gitignore.
- El script auth_whoop.py contiene CLIENT_SECRET hardcodeado —
  es solo para reautenticación manual, no se ejecuta en producción.
- El wrapper de gcalcli usa variables de entorno para las credenciales
  (GCALCLI_CLIENT_ID / GCALCLI_CLIENT_SECRET) — no hardcodeadas.
- GitHub Push Protection está activo en el repo: bloqueará commits
  con secretos detectados automáticamente.
